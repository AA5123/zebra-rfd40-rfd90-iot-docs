get_operating_mode

Command: get_operating_mode

Description

This command is designed to fetch the current operating mode information of RFID device within the IOTC system

Request

Request Fields

FieldTypeRequiredDescription
commandstring (enum)YesSpecifies the name of the command executed. In this case, it is get_operating_mode to fetch the current operating mode information of the device.
requestIdstringYesA unique identifier assigned to the request, used to track and correlate responses with their corresponding requests.

operatingMode.operatingModes.tagMetaDataToEnable.XPCbooleanNoIndicates whether to report the Extended Protocol Control (XPC) word. XPC provides additional protocol-related information for advanced tags, such as custom memory configurations.
operatingMode.operatingModes.tagMetaDataToEnable.CRCbooleanNoIndicates whether to report the CRC value. CRC is used to verify the integrity of the tag's data and ensure error-free communication.
operatingMode.operatingModes.tagMetaDataToEnable.EPCbooleanNoIndicates whether to report the EPC. EPC is the unique identifier stored in the tag's memory, typically used for item identification.
operatingMode.operatingModes.tagMetaDataToEnable.TIDbooleanNoIndicates whether to report the TID. TID is a factory-programmed, unique identifier for the tag, often used for authentication or anti-counterfeiting.

operatingMode.operatingModes.tagMetaDataToEnable.MACbooleanNoIndicates whether to report the MAC address of the reader. This is useful for identifying the specific reader that performed the tag read in networked environments.
operatingMode.operatingModes.tagMetaDataToEnable.HOSTNAMEbooleanNoIndicates whether to report the hostname of the reader. Hostnames are helpful in networked environments where readers are identified by their assigned names.
