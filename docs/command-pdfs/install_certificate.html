ZEBRA TECHNOLOGIES install_certificate — RFD40 / RFD90 IoT Connector API Reference

install_certificate

Command: install_certificate

Description

This API is used to install certificates.

Request

Request Fields

FieldTypeRequiredDescription
commandstringYesCommand issued to install new certificate
requestIdstringYesA unique identifier for the request, allowing tracking and debugging of the operation.
certDetailsobjectNo
certDetails.namestringNoThe name of the certificate, typically representing the application or agent.
certDetails.typestring (enum)YesThe type of the certificate, indicating its purpose or usage (e.g., client, server, etc.).
certDetails.authenticationTypestring (enum)YesSpecifies the authentication method required. If BASIC, username and password are required. If NONE, no authentication credentials are needed.
certDetails.urlarrayNoA list of key-value pairs representing URL parameters.
certDetails.authenticateOptionsobjectNoOptions for authentication, including username and password.
certDetails.authenticateOptions.usernamestringNoThe username for authentication when required.
certDetails.authenticateOptions.passwordstringNoThe password for authentication when required.
certDetails.pfxPasswordstringNoThe password for the PFX (Personal Information Exchange) file used in secure communications.
certDetails.headerParamsarrayNoA list of key-value pairs representing HTTP header parameters.
certDetails.queryParamsarrayNoA list of key-value pairs representing query parameters.
certDetails.verificationTypestring (enum)NoSpecifies the type of verification to perform, such as peer verification or host verification.
certDetails.caCertificateFileContentstringNoThe content of the CA (Certificate Authority) certificate file, used for verifying identities.
