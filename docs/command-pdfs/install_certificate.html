install_certificate

Command: install_certificate

Description

This API is used to install certificates.

Download pdf: 📄 [Download install_certificate as PDF](https://aa5123.github.io/zebra-rfd40-rfd90-iot-docs/command-pdfs/install_certificate.pdf)

Request

Request Fields

FieldTypeRequiredDescriptionExample
commandstringYesCommand issued to install new certificateinstall_certificate
requestIdstringYesA unique identifier for the request, allowing tracking and debugging of the operation.abc123
certDetailsobjectNo
certDetails.namestringNoThe name of the certificate, typically representing the application or agent.sotiAgent
certDetails.typestring (enum)YesThe type of the certificate, indicating its purpose or usage (e.g., client, server, etc.).client
certDetails.authenticationTypestring (enum)YesSpecifies the authentication method required. If BASIC, username and password are required. If NONE, no authentication credentials are needed.BASIC
certDetails.urlarrayNoA list of key-value pairs representing URL parameters.
certDetails.authenticateOptionsobjectNoOptions for authentication, including username and password.
certDetails.authenticateOptions.usernamestringNoThe username for authentication when required.rfid
certDetails.authenticateOptions.passwordstringNoThe password for authentication when required.rfid
certDetails.pfxPasswordstringNoThe password for the PFX (Personal Information Exchange) file used in secure communications.pass@123
certDetails.headerParamsarrayNoA list of key-value pairs representing HTTP header parameters.
certDetails.queryParamsarrayNoA list of key-value pairs representing query parameters.
certDetails.verificationTypestring (enum)NoSpecifies the type of verification to perform, such as peer verification or host verification.NONE
certDetails.caCertificateFileContentstringNoThe content of the CA (Certificate Authority) certificate file, used for verifying identities.
