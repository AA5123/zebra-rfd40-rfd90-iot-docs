ZEBRA TECHNOLOGIES set_os — RFD40 / RFD90 IoT Connector API Reference

set_os

Command: set_os

Description

Command issued to set new firmware & initiate firmware update

Request

Request Fields

FieldTypeRequiredDescription
commandstringYesSpecifies the operation being performed, in this case, setting the reader operating system.
requestIdstringYesA unique identifier for the request, allowing tracking and debugging of the operation.
OSUpdateDetailsobjectNo
OSUpdateDetails.urlstringYesThe URL where the OS update can be downloaded. Must be a valid URI.
OSUpdateDetails.authenticationTypestring (enum)YesSpecifies the type of authentication required for accessing the URL. NONE - No authentication is needed. BASIC - Username and password are required. CERTIFICATE - A valid inline certificate or preinstalled certificate is required. TOKEN - Token-based authentication is required.
OSUpdateDetails.authOptionsobjectNoContains authentication options such as username and password for BASIC authentication.
OSUpdateDetails.authOptions.usernamestringNoThe username for BASIC authentication.
OSUpdateDetails.authOptions.passwordstringNoThe password for BASIC authentication.
OSUpdateDetails.headerParamarrayNoA list of custom header parameters to include in the HTTP request.
OSUpdateDetails.queryParamarrayNoA list of query parameters to include in the URL.
OSUpdateDetails.verificationTypestring (enum)NoSpecifies the type of verification to perform during the connection. NONE - No verification. VERIFY_PEER - Verify the peer's certificate. VERIFY_HOST - Verify the host's certificate. VERIFY_HOST_PEER - Verify both the host and peer certificates.
OSUpdateDetails.caCertificateFileContentstringNoThe content of the CA certificate in PEM format, required for certificate-based authentication.
OSUpdateDetails.caCertificateFilestringNoThe file path to the CA certificate, required for certificate-based authentication.

Response

Response Fields

FieldTypeRequiredDescription
commandstringYes
requestIdstringYes
apiVersionstring (enum)Yes
responseobjectNoStatus of the command execution — check code to confirm success (0) or identify the error.
response.codeintegerYesResponse code indicating success or failure.
  • 0 — Success
  • 1 — Command payload is accepted
  • 4 — Firmware update in progress
  • 8 — Insufficient flash size
  • 9 — File not found
  • 13 — Firmware update Failed
  • 14 — Battery is low, Cannot update firmware
response.descriptionstringYesHuman-readable description of the response code (e.g. "Success", "Invalid payload").
