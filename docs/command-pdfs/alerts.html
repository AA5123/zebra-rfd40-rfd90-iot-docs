ZEBRA TECHNOLOGIES alerts — RFD40 / RFD90 IoT Connector API Reference

alerts

Command: alerts

Description

Represents a system-generated alert or notification event. Note: Some event types such as antenna, temperature, usage, GPI, and user app info are not supported.

Request

Request Fields

No fields available.

Request Examples

No examples available.

Response

Response Fields

FieldTypeRequiredDescriptionExample
typestring (enum)YesSpecifies the type of alert or message being generated, such as an event, notification, exception, or alert.
timestampstringYesIndicates the time at which the alert was generated, formatted as a time string.22:54:25
statestring (enum)YesRepresents the state of the alert. 'SET' indicates an active alert, 'CLEAR' indicates the alert has been resolved, and 'ONESHOT' is a one-time alert.
idstring (enum)YesA unique identifier for the alert type, such as a low battery warning or a temperature event.BATTERY_LOW
prioritystring (enum)YesDefines the priority level of the alert, ranging from critical to low.CRITICAL
alertDetailsobjectNoContains additional details about the alert, which may vary depending on the specific alert type.
