ZEBRA TECHNOLOGIES set_post_filter — RFD40 / RFD90 IoT Connector API Reference

set_post_filter

Command: set_post_filter

Description

This operation is used to set a post-filter for RFID device, enabling the filtering of tags scanned by the device based on specific criteria.

Request

Request Fields

FieldTypeRequiredDescription
commandstringYesSpecifies the name of the command to be executed. In this case, it is set_post_filter.
requestIdstringYesA unique identifier for the request. This ensures traceability and prevents duplicate processing of the same request.
postFilterPayloadobjectNo
postFilterPayload.operationstring (enum)NoSpecifies the operation to perform on the filter such as add a new filter, remove and update an existing filter.
postFilterPayload.dataEpTypestring (enum)NoSpecifies the data endpoint to which the filter should be applied. Two data endpoints are supported for reporting filtered tag data.
postFilterPayload.matchPatternstringNoDefines the value to match for the filter. For prefix and suffix filters, only hexadecimal digits are allowed, and the number of digits must be even. When a prefix filter is used, select filters cannot be applied.
postFilterPayload.matchPatternMethodstring (enum)NoSpecifies the method used to match the tag ID. PREFIX: Matches the beginning of the tag ID. SUFIX: Matches the end of the tag ID. REGEX: Matches the tag ID using a regular expression.
postFilterPayload.reportOperationstring (enum)NoSpecifies the filter operation to apply. INCLUDE: Only tags matching the filter criteria will be reported. EXCLUDE: Tags matching the filter criteria will be excluded from reporting.
postFilterPayload.reportDurationintegerNoSpecifies the duration (in seconds) to wait before reporting a tag again after it has already been reported. The filter ensures that as long as the tag is being read by the reader, it will not be reported again until the specified duration has elapsed.
postFilterPayload.seenCountintegerNoSpecifies the number of times a tag must be detected before its data is sent out. This helps in filtering out tags that are inconsistently read.
postFilterPayload.tagListobjectNo
postFilterPayload.tagList.operationstring (enum)NoSpecifies whether the tag should be blacklisted or whitelisted. BLACKLIST: Excludes tags matching the patterns from reporting. WHITELIST: Includes only tags matching the patterns for reporting.
postFilterPayload.tagList.tagPatternsarrayNoAn array of tag patterns to be used for whitelisting or blacklisting. Multiple patterns can be provided to define the filter criteria.
postFilterPayload.rssiThresholdintegerNoSpecifies the RSSI threshold (in dBm). Tags with an RSSI below this threshold will be ignored by the reader. The value must be between -88 dBm (minimum sensitivity) and -26 dBm (maximum sensitivity).

Request Examples

Set post filter 1

{
  "command": "set_post_filter",
  "requestId": "abc123",
  "postFilterPayload": {
    "operation": "ADD",
    "dataEpType": "DATA_EP1",
    "matchPattern": "FFFFBBBBA500",
    "matchPatternMethod": "PREFIX",
    "reportOperation": "INCLUDE",
    "reportDuration": 0,
    "seenCount": 0,
    "rssiThreshold": -88
  }
}

Set post filter 2

  "command": "set_post_filter",
  "requestId": "abc123",
  "postFilterPayload": {
    "operation": "ADD",
    "dataEpType": "DATA_EP1",
    "matchPattern": "E280",
    "matchPatternMethod": "REGEX",
    "reportOperation": "EXCLUDE",
    "reportDuration": 0,
    "seenCount": 0,
    "rssiThreshold": -88
  }
}
